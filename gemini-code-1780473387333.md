## 1. Asyncio Subprocess Deadlock Resolution
**CRITICAL DATA ANOMALY:** External recon scripts generated output that saturated the OS pipe buffer. Using `process.wait()` or blocking synchronous reads caused total event loop freezing and crashed the local C2 dashboard.
* **Directive:** ALL asynchronous external script executions utilizing `stdout=asyncio.subprocess.PIPE` MUST use `await process.communicate()`. 
* **Implementation:** The `communicate()` method concurrently drains standard output and standard error buffers, preventing host node storage and memory deadlocks. Never use `process.wait()` for OSINT bridge execution.

## 2. JARM Fingerprinting & Passive Intelligence
**CRITICAL SECURITY RISK:** Active TLS fingerprinting via native Python `asyncio` sockets possesses a highly identifiable signature. Modern WAFs and CDNs will flag and block the primary host IP.
* **Directive:** Do NOT develop or execute `osint_jarm_bridge.py`. Active TLS probing is strictly prohibited to maintain OPSEC.
* **Implementation:** JARM/JA3 hashes must be extracted passively. Modify `osint_shodan_bridge.py` and `osint_zoomeye_bridge.py` to parse and extract the JARM fields directly from their existing API JSON responses.

## 3. Discord Visual Payload Pipeline (URLScan)
* **Directive:** Do not store URLScan visual artifacts locally on the primary node. Prevent D:\ drive bloat.
* **Implementation:** 1. `osint_urlscan_bridge.py` extracts the remote image link: `https://urlscan.io/screenshots/{scan_id}.png`.
  2. `watchtower_gui.py` appends this URL directly to the Discord webhook JSON payload: `"image": {"url": image_url}`.
  3. Discord natively renders the remote image embed without requiring local binary storage.
