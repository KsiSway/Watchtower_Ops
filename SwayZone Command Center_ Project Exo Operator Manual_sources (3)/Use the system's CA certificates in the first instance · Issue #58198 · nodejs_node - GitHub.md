What steps will reproduce the bug?

Make an HTTPs request with a CA that exists in the system trust store, but not in the node trust store.

How often does it reproduce? Is there a required condition?

What is the expected behavior? Why is that the expected behavior?

The call should succeed as the system trust store should be respected.

What do you see instead?

The call fails because Node packages up its own CA certs and ignores the system trust store.
