Local agents in Visual Studio Code

Local agents run interactively within Visual Studio Code on your machine. They work on your current workspace and have access to the full range of tools and models available in VS Code, including extension-provided tools and MCP servers. By

creating custom agents

https://code.visualstudio.com/docs/copilot/customization/custom-agents

, you can let the agent assume a specific role or persona for a task, such as a code reviewer, tester, or documentation writer.

Local agents operate in the chat interface in VS Code. When you close a chat session, the local agent remains active and you can track it in the sessions view.

Why use local agents?

https://code.visualstudio.com/docs/copilot/agents/local-agents#\_why-use-local-agents

Interactive conversations that require immediate feedback, such as brainstorming, planning, or tasks that aren't yet fully defined

Tasks that require context from your developer environment, such as linting errors, stack traces, unit test results

Tasks that require access to specific tools from VS Code extensions or MCP servers or need to use specific models like BYOK models

Tasks that don't require collaboration from other team members

Key characteristics

https://code.visualstudio.com/docs/copilot/agents/local-agents#\_key-characteristics

Runs within VS Code on your local machine and works on your current workspace

Interactive chat-based interface for real-time feedback and iteration

Full access to your workspace, files, and context

Can use all models available to you in VS Code, including BYOK models and models from other providers

Does the language model run locally on my machine?

The location of the language model does not depend on the agent type. The agent type determines where the agent harness is running, so where the agent orchestration logic runs. For example, local agents run within VS Code on your machine, while Copilot cloud agents run on remote servers.

The location of the language model depends on the model provider you choose for a request. For example, if you choose a model provided by GitHub Copilot, it will run on remote infrastructure. If you choose a

https://code.visualstudio.com/docs/copilot/customization/language-models#\_bring-your-own-language-model-key

that connects to a model running on your machine or private infrastructure, then the model will run there.
