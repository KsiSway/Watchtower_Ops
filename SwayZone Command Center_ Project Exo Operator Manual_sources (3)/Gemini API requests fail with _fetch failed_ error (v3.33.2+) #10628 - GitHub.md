I've analyzed this issue and the related issues

https://github.com/RooCodeInc/Roo-Code/issues/9465

https://github.com/RooCodeInc/Roo-Code/issues/10508

. Here's my proposed implementation plan:

The error "TypeError: fetch failed" is a generic wrapper from the

@google/genai

#### SDK that obscures the actual underlying cause. This could be due to:

DNS resolution failures

Proxy/firewall configurations not being respected by native Node.js fetch

Request timeouts (the SDK may have strict defaults)

SSL/TLS certificate validation issues

Network connectivity issues specific to certain environments

The issue affects users in various network configurations, and the current error handling provides no diagnostic information to help them troubleshoot.

Proposed Plan

Extract and preserve the underlying error cause

: When catching errors in the Gemini handler, check for

error.cause

which often contains the actual fetch failure reason (e.g., ECONNREFUSED, ETIMEDOUT, DNS lookup failures). Include this in the error message so users can diagnose the issue.

Add timeout configuration

: Pass a configurable timeout value via

httpOptions.timeout

@google/genai

client. This prevents silent/unclear failures on slow network connections and gives users control over timeout behavior.

Improve error message formatting

: Update the error handling to provide actionable diagnostic information, including:

The underlying error cause (if available)

Suggestions for common issues (proxy settings, network connectivity, API key validation)

Use the existing error handler utility

: Leverage the

handleProviderError

function for consistent error handling across providers, ensuring HTTP status codes and error details are preserved.

#### This approach will:

Help users understand why their requests are failing

Provide actionable information for troubleshooting

Maintain consistency with other provider error handling patterns

Please mention

https://github.com/roomote

in a comment saying that you approve this plan, and I'll proceed with the implementation.
